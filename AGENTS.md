# Repository Rules

## GitHub公開フロー

- ローカルの変更をGitHubへ公開するときは、ブランチ作成・ステージ・コミット・pushに`git`を使用する。
- Pull Requestの作成・更新には、認証済みのGitHub CLI（`gh`）を使用する。
- ChatGPT/CodexのGitHub連携は読み取り・参照用途として扱い、Pull Request作成には依存しない。
- Pull Requestは、ユーザーが明示的にReadyを指定しない限りDraftで作成する。
- Pull Request本文は「概要」「作業内容の詳細」「検証内容」「その他、補足事項など」の構成にする。
