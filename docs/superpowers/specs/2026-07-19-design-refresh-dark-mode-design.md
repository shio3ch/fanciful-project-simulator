# デザイン刷新（ドラマ演出型）+ ナイトモード 設計書

日付: 2026-07-19
ステータス: 承認済み（実装計画待ち）

## 1. 背景と目的

現状のUIは「白カード + slate + indigo」の無加工Tailwindスタイルで、コンテンツ（炎上・奇跡の復活といったプロジェクトドラマ）に対して見た目が淡白。以下を目的にビジュアルレイヤーを刷新する。

- **ドラマ演出型デザイン**: プロジェクトステータスに連動した色演出で「物語」を感じさせる
- **ナイトモード対応**: ライト / ダーク / システム追従の3状態切替
- 機能・情報設計（セクション構成、データフロー）は変更しない。見た目のレイヤーのみ刷新する

演出の強さは「中」: ヒーロー部（ProjectSummary）の背景と画面内の差し色がステータスに連動する。画面全体のトーンまでは変えない。

## 2. デザイントークン基盤

`src/index.css` に Tailwind v4 の `@theme` + CSS変数でセマンティックトークンを定義する。

### 2.1 セマンティックトークン

| トークン | 用途 | ライト | ダーク（方向性） |
|---|---|---|---|
| `--color-surface` | ページ背景 | slate-50相当 | 深いネイビー系（例: slate-950近辺） |
| `--color-card` | カード背景 | 白 | surfaceよりわずかに明るい階調 |
| `--color-card-raised` | 吹き出し・ネストカード | slate-50相当 | cardよりさらに一段明るい |
| `--color-ink` | 本文 | slate-900相当 | slate-100相当 |
| `--color-ink-muted` | 補助文字 | slate-500相当 | slate-400相当 |
| `--color-line` | 境界線 | slate-200相当 | 明度低めのライン |
| `--color-accent` | 現在ステータス連動の差し色 | ステータスごと | 同左（ダーク用に彩度調整） |

- コンポーネント内の `bg-white` / `text-slate-900` 等の直書きを全てトークン参照（`bg-card` `text-ink` 等）へ置換する
- コンポーネント側には `dark:` バリアントを書き散らさない。トークンの値だけがテーマで変わる

### 2.2 ステータス → アクセント色マッピング

`src/lib/ui.ts` に集約する。7つの `ProjectStatus` それぞれに hue を割当てる。

| ステータス | hue |
|---|---|
| 順調 | emerald |
| 少し怪しい | lime |
| 黄色信号 | amber |
| 炎上中 | red-orange |
| 崩壊寸前 | red |
| 奇跡の復活 | sky |
| 無事リリース | emerald |

現在スナップショットのステータスから `--accent`（および淡色版 `--accent-soft` 等）をルート要素にインラインCSS変数として注入し、以下が連動する。

- ProjectSummary の背景グラデーションとステータスバッジ
- タイムラインの現在章ハイライト
- HealthCheck チャートの ReferenceLine（現在位置線）

各 hue はライト / ダーク双方で可読なペア（背景・文字・ボーダー・グラフ線）を定義する。

## 3. ナイトモード

- **切替UI**: ヘッダーに 🌙/☀️ ボタン。ライト → ダーク → システム追従 の3状態トグル
- **保存**: localStorage（既存 `storage.ts` の流儀に合わせたキー設計）。システム追従時は `prefers-color-scheme` を監視
- **適用方式**: `<html data-theme="dark">` の付け外し + `@custom-variant dark` でトークン値を切替
- **FOUC防止**: `index.html` の `<head>` にインライン初期化スクリプトを置き、初回描画前に `data-theme` を確定する
- **ダークのトーン**: 「夜のオフィス」。純黒ではなく深いネイビー系。カードは背景よりわずかに明るくして浮かせる。炎上系の赤はダーク上でも映える彩度に調整

## 4. コンポーネント別の変更

### 4.1 Header
- テーマ切替ボタンを追加
- 既存の生成・設定ボタンはトークン化のうえ現行踏襲

### 4.2 ProjectSummary（ヒーロー化）
- プロジェクト名を `text-2xl`〜`text-3xl` の大見出しへ
- ステータスバッジを絵文字付きの大きめピルへ
- カード背景にステータス色の淡いグラデーション（例: 炎上中は上部にじわっと赤み）
- 顧客 / 期間 / 予算 / 難易度 / 現在工程 は「スタットチップ」の横並びに整理

### 4.3 Timeline（章仕立て）
- 各エピソードカードを「第n章」表記にし、カード間を進行ラインで接続
- 現在章はアクセント色でハイライト、過去章は通常、未来章は「未読」風に淡く
- セクションの枠カードをやめ、全幅の帯として配置（物語の背骨として強調）

### 4.4 MemberList（カードゲーム風味）
- 絵文字アバターを円形バッジ化
- ステータスバーをグラデーション化
- 特殊スキルを「スキルチップ」として強調

### 4.5 EpisodeDetail（読み物化）
- 見出しを大きく、階層を明確に
- チャット抜粋を吹き出しUIへ
- KPI変化を ▲▼ 付きチップへ

### 4.6 チャート / 相関図（ダーク対応）
- Recharts（HealthCheck / BurndownChart）の16進数直書き色をトークン由来に変更。軸・ツールチップもテーマ対応
- xyflow（RelationshipMap）のエッジ色 `REL_COLORS` にダーク用バリアントを用意
- CSS変数の値はレンダリング時に取得してチャートライブラリへ渡す（テーマ切替時に再取得）

### 4.7 その他（HealthCheck バー、Retrospective、SettingsModal、GeneratingOverlay、エラー表示）
- トークン化 + 上記と同じ意匠（角丸・影・チップ）に揃える

## 5. タイポグラフィ

- 日本語Webフォントは追加しない（重量対効果が悪い）。システムフォントスタックのままウェイト（bold/semibold）とサイズ・トラッキングの抑揚で階層を作る
- 新規ライブラリの追加もなし。Tailwind v4 の機能のみで実装する

## 6. テスト

- テーマの保存 / 復元 / 3状態遷移ロジックをユニットテスト（`storage.test.ts` の流儀）
- 「ステータス → アクセント色」マッピングの網羅テスト（`ui.test.ts` に追加。7ステータス全てに定義があること）
- 見た目の確認は `npm run dev` での目視（ライト / ダーク両方、代表的なステータスのサンプル切替）

## 7. スコープ外

- セクションの並び・情報構成の変更
- 新機能（アニメーション演出の大掛かりなもの、効果音等）
- 「強」演出（画面全体のトーン変化）。将来やる場合もトークン基盤の上に載せられる
